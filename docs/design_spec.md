指定された要件（{{$json.content}}）が展開されていないため、設計書を生成できません。  
まず要件JSONの提供をお願いします。あるいは、以下の必須・推奨項目へ回答いただければ、それをもとに設計書を作成します。

---  
お願いしたい要件項目（JSONで提供する場合はこれらのキーを含めてください）

必須項目
- プロジェクト名
- 対象資産クラス（例：FX、株式、仮想通貨、先物）
- 対象市場／取引所（例：NYSE、Binance、BitMEX）
- 戦略タイプ（例：裁定、トレンドフォロー、マーケットメイキング、スキャルピング）
- 注文頻度（例：低頻度：1日数回、HFT：ミリ秒単位）
- 注文の発注方法（成行／指値／IOCなど）
- 資金管理ルール（例：1トレードあたりの最大リスク、レバレッジ、ポジション上限）
- リスク管理ルール（例：最大ドローダウン、連続損失で止める閾値）
- バックテスト/フォワードテスト要件（必要/不要、期間）
- 実行環境（クラウド／オンプレ、OS要件）
- 可用性要件（稼働率、RTO/RPO）
- レイテンシ要件（平均/99パーセンタイル）
- 規制・コンプライアンス要件（顧客資産分離、ログ保存期間等）
- 監視・アラート要件（通知チャネル、閾値）
- 認証・権限管理（APIキー管理、ロール）
- データ保持期間（ティック、板情報、約定履歴、ログ）
- 開発言語の希望（例：Python/Go/Java/C++、無ければ提案可）
- CI/CD / デプロイ方法の希望

推奨項目（あるとより詳細に設計できます）
- 取引手数料やスリッページ想定
- 注文キャンセル/再送ポリシー
- 市場データの購読方法（WebSocket/REST、深さ）
- マルチアセットやマルチアカウントの有無
- 長期運用での拡張（スケーリング）要件
- セキュリティ要件（HSM、KMS、VPN）
- テストカバレッジや品質基準

---

もし明示的な要件がまだ用意できない場合は、以下の「仮定」を使って設計書を作成できます。続行する場合は「仮定で作成して」と返信してください。仮定例は下記のとおりです。

仮定（デフォルト）
- 資産クラス：仮想通貨（スポット）
- 取引所：Binance（スポット）とCoinbase Pro（片方は実装例）
- 戦略：トレンドフォロー（移動平均クロス）と簡易リスク管理（ATRベースのストップ）
- 注文頻度：中頻度（分〜秒単位）
- 注文タイプ：成行/指値、IOCは必須ではない
- バックテスト：過去2年分のティック/1分足で実施
- 実行環境：AWS（ECS/Fargate + RDS + S3 + CloudWatch）
- 可用性：99.9% 程度、フェールオーバー対応
- レイテンシ：平均100ms未満を目標
- 開発言語：Python（戦略・バックテスト）＋Go（低レイテンシ注文ブローカー）
- 認証：APIキー暗号化、Vault/KMSで管理
- 監視：Prometheus + Grafana、Slack/メール通知
- データ保持：ティック1年、OHLC/集計データ無期限保存はS3
