要件整理の内容がプレースホルダ「{{$json.content}}」になっており、実際の要件が参照できません。設計書を正確に作成するため、以下のいずれかを選んでください。

A) 実際の要件JSON（または箇条書き）を送る  
   推奨フィールド（可能ならこの形式でください）：
   - 対象市場（例：FX, 株式, 仮想通貨）と取引所/ブローカー名  
   - 取引対象（例：通貨ペア, 銘柄）  
   - 注文頻度／レイテンシ要件（例：ミリ秒級、分次、日次）  
   - サポートする注文種類（成行、指値、OCO 等）  
   - 注文数量・ポジション制約、リスクルール（最大ドローダウン、最大ポジション等）  
   - 実行環境（クラウド：AWS/GCP/Azure、オンプレ等）  
   - 言語・フレームワークの指定（例：Python, Go, Java）  
   - データ保持期間・DB要件（履歴・約定・ログ）  
   - バックテスト・フォワードテスト要件  
   - 可用性・復旧要件（RTO/RPO、SLA）  
   - モニタリング/アラート要件（メール/Slack/PagerDuty）  
   - 認証・権限・監査要件（複数ユーザ、ロール）  
   - 規制／コンプライアンス要件（KYC、監査ログ等）  
   - その他（UIの有無、運用体制、スケジュール、予算など）

B) こちらで一般的な自動売買システム設計の想定要件を定めて作成する（標準的な設計書を作成）  
   - 例：仮想通貨とFXを対象に、分〜秒単位の戦略を想定。言語はPython（戦略）＋Go（マッチング/オーダー管理）。AWS上で可用性を確保。バックテスト・シミュレーション機能有り。  

どちらにしますか？Bを選ぶ場合、想定してほしいポイント（対象市場、レイテンシ、言語、クラウド等）を1〜2行で教えてください。提供いただければ、指定のフォーマット（Markdown、日本語）で設計書（1〜10項目）を作成します。
